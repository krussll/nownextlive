# COPY_GUIDELINES.md

## Purpose
This guide defines rules for writing consistent, friendly, non‑technical text across NowNext.

## General Rules
- Short sentences.
- No jargon.
- Use everyday words.
- Speak directly to the user.
- Prefer clarity over cleverness.

## Buttons
- Always verb-led.
- Keep to 1–3 words.
Examples:
- “Try demo”
- “Update screen”
- “Create board”

## Labels & Form Fields
- Be literal and plain.
Examples:
- “Now”
- “Next”
- “Room name”

## Tooltips
- Keep to one sentence.
- Explain benefit, not mechanism.

## Error Messages
- Friendly and calm.
- Never blame the user.
Examples:
- “Something didn’t load. Try again.”
- “Connection lost — reconnecting.”

## Help Text
- Practical instructions.
- Prefer numbered steps.
